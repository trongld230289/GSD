## Deviations from Plan

### Structural Differences (implementation evolved beyond bootstrap plan)

**1. Store architecture diverged: single useStore.ts vs. two files**
- **Plan specified:** `src/stores/authStore.ts` + `src/stores/txStore.ts`
- **Actual:** `src/store/useStore.ts` (single file, three named exports)
- **Rationale:** Simpler import pattern; avoids circular dependency risk between stores
- **Impact:** None — same functionality, different file layout

**2. GAS URL hardcoded instead of VITE_GAS_URL env var**
- **Plan specified:** `VITE_GAS_URL` in `.env.local`
- **Actual:** URL literal in `src/api/gas.ts`
- **Rationale:** GitHub Pages deploy via Actions doesn't need secrets management for public GAS URL
- **Impact:** None — URL is not sensitive; app works identically

**3. utils/id.ts omitted — IDs generated server-side**
- **Plan specified:** `crypto.randomUUID()` client-side ID generation
- **Actual:** GAS `Utilities.getUuid()` generates IDs on the server
- **Rationale:** Server-generated IDs are more reliable; no optimistic ID conflicts
- **Impact:** Positive — eliminates optimistic update complexity

**4. categories.ts in `data/` vs. `constants/` with extended schema**
- **Plan specified:** `src/constants/categories.ts` with 16 entries matching plan schema
- **Actual:** `src/data/categories.ts` with CATEGORY_META[] (16 entries + description field)
- **Impact:** None — same category data, richer type
