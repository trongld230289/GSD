## Authentication & Identity

**Auth Provider:**
- Google OAuth 2.0 via Google Identity Services

**Implementation Details:**
- Flow: User clicks Google sign-in → GSI returns JWT credential → App decodes JWT payload locally
- Token storage: ID token stored in auth store (`idToken` in `useAuthStore`)
- User data extracted: `email`, `name`, `picture`, `sub` from JWT payload
- Session persistence: Zustand persist middleware keeps auth state across page reloads
- Logout: `clearUser()` action clears auth store (no server-side session)

**Protected Routes:**
- `/login` - Only accessible without authentication (redirects to `/` if authenticated)
- `/` (HomePage) - Requires authentication (redirects to `/login` if not authenticated)
- `/reports` (ReportsPage) - Requires authentication (redirects to `/login` if not authenticated)
