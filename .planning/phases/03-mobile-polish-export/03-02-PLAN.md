Fix the iPhone safe area gap: the bottom navigation bar is currently obscured by the iOS home indicator swipe bar when the app is installed as a PWA. Output: Updated `index.html` and `BottomNav.tsx`. No new files. Build must stay clean.

  <name>Task 1: Fix viewport meta and BottomNav safe-area padding</name>
  <files>finance-tracker/index.html, finance-tracker/src/components/BottomNav.tsx</files>
  <action>
    TWO changes in this task:

    **1. finance-tracker/index.html — fix viewport meta tag:**

    Replace the current viewport meta:
    ```html
    <meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no" />
    ```
    With:
    ```html
    <meta name="viewport" content="width=device-width, initial-scale=1.0, viewport-fit=cover" />
    ```
    - Remove `maximum-scale=1.0, user-scalable=no` — fails WCAG 1.4.4 and Lighthouse accessibility audit
    - Add `viewport-fit=cover` — required for iOS to expose `env(safe-area-inset-bottom)` to CSS

    Also change the apple-mobile-web-app-status-bar-style meta from `default` to `black-translucent`:
    ```html
    <meta name="apple-mobile-web-app-status-bar-style" content="black-translucent" />
    ```
    This makes the iOS status bar overlay the green header (transparent) rather than pushing it down. Goes together with `viewport-fit=cover`.

    **2. finance-tracker/src/components/BottomNav.tsx — add safe-area padding:**

    The `.safe-pb` class is already defined in index.css as `padding-bottom: env(safe-area-inset-bottom)`. Use an inline style instead of the class to be explicit and avoid any Tailwind purge concern:

    Change the nav element from:
    ```tsx
    <nav className="fixed bottom-0 left-0 right-0 max-w-md mx-auto bg-white border-t border-gray-100 z-30 flex">
    ```
    To:
    ```tsx
    <nav
      className="fixed bottom-0 left-0 right-0 max-w-md mx-auto bg-white border-t border-gray-100 z-30 flex"
      style={{ paddingBottom: 'env(safe-area-inset-bottom)' }}
    >
    ```

    Leave the NavLink `py-2` classes unchanged — the nav-level `paddingBottom` adds to the bottom of the nav bar itself (below the tab buttons), pushing the whole nav up above the home indicator. The `py-2` padding inside each tab item is separate and stays.

    Do NOT modify BottomNav routing, tab labels, or icons. This is a single-line CSS change.
